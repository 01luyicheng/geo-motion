## 待处理问题

### 高优先级

#### 1. 测试覆盖率为零
- **文件**: 全局
- **问题描述**: 项目没有任何测试文件，存在较高的维护风险
- **风险等级**: 高
- **影响**: 每次修改都需要手动回归测试，重构困难
- **建议**: 
  - 添加 API 路由单元测试
  - 添加工具函数测试
  - 推荐使用 Vitest + Testing Library

#### 2. 未使用的依赖
- **文件**: `frontend/package.json`
- **问题描述**: 以下依赖已安装但未使用，增加约 200KB 包体积
  - `@tanstack/react-query`
  - `framer-motion`
  - `fabric`
- **风险等级**: 中
- **建议**: 执行 `npm uninstall @tanstack/react-query framer-motion fabric`

#### 3. 缺少错误边界组件
- **文件**: `frontend/src/app/`
- **问题描述**: 无 Error Boundary 组件，运行时错误可能导致白屏
- **风险等级**: 中
- **建议**: 添加全局错误边界组件

#### 4. 缺少环境变量文档
- **文件**: `frontend/src/lib/openrouter.ts:68`
- **问题描述**: 使用 `NEXT_PUBLIC_BASE_URL` 环境变量，但项目缺少 `.env.example` 或相关文档说明
- **风险等级**: 低
- **影响**: 新开发者不清楚需要配置哪些环境变量
- **建议**: 添加 `.env.example` 文件，列出所有需要配置的环境变量

#### 5. 生产环境日志泄露敏感信息
- **文件**: `frontend/src/lib/openrouter.ts` 多处
- **问题描述**: 大量 `console.log` 输出请求体、响应内容等
- **风险等级**: 中
- **影响**: 生产环境可能泄露 API 密钥、用户数据
- **建议**: 使用环境变量控制日志级别，生产环境禁用详细日志

### 中优先级

#### 6. 单文件过大
- **文件**: 
  - `frontend/src/app/page.tsx` (427行)
  - `frontend/src/app/analyze/[id]/page.tsx` (450行)
- **问题描述**: 单文件职责过多，难以维护
- **风险等级**: 中
- **建议**: 
  - 提取 hooks（如 `useStreamRequest`、`useAnimation`）
  - 拆分为更小的组件

#### 7. SSE 流处理性能问题
- **文件**: `frontend/src/app/page.tsx:110`
- **问题描述**: 每次 chunk 都调用 `setStreamContent`，高频更新可能导致 UI 卡顿
- **风险等级**: 中
- **建议**: 使用防抖或批量更新

#### 8. 外部服务依赖风险
- **问题描述**: 
  - GeoGebra 依赖 CDN 加载，无本地 fallback
  - OpenRouter 单一 VLM 提供商，无备用方案
- **风险等级**: 中
- **建议**: 
  - 添加 GeoGebra 加载失败的优雅降级
  - 考虑支持多 VLM 提供商

#### 9. XSS 安全风险
- **文件**: `frontend/src/app/page.tsx:367`
- **问题描述**: 流式内容直接渲染，VLM 返回恶意内容可能被执行
- **验证结果**: 流式内容在 `<pre>` 标签中渲染，**没有使用** `dangerouslySetInnerHTML`，React 默认会转义 HTML，当前实现是安全的
- **风险等级**: 低（实际安全）

#### 10. 图片验证不足
- **文件**: `frontend/src/components/ImageUploader.tsx:37`
- **问题描述**: 仅检查 MIME 类型，可被伪造
- **风险等级**: 低
- **说明**: 服务端有额外校验，风险较低
- **建议**: 添加文件头魔数验证（可选）

### 低优先级

#### 11. 类型定义不完整
- **文件**: `frontend/src/types/index.ts:61-67`
- **问题描述**: `window.ggbApplet` 类型定义缺少部分方法声明
- **风险等级**: 低
- **状态**: 当前功能够用，但不完整

#### 12. 缺少代码格式化工具
- **问题描述**: 缺少 Prettier 配置，代码风格不统一
- **风险等级**: 低
- **建议**: 添加 Prettier + ESLint 增强规则

#### 13. 日志格式不统一
- **问题描述**: 
  - `analyze/route.ts` 有详细日志
  - `generate-graphic/route.ts` 日志简略
- **风险等级**: 低
- **建议**: 统一日志格式和级别

#### 14. 硬编码配置
- **文件**: `frontend/src/lib/openrouter.ts:7`
- **问题描述**: `MODEL = 'bytedance-seed/seed-2.0-mini'` 硬编码
- **风险等级**: 低
- **建议**: 移至环境变量或配置文件

#### 15. 类型断言过多
- **位置**: 多处使用 `as unknown as` 和 `as` 断言
- **示例**: `GeoGebraViewer.tsx:151`
- **问题描述**: 过度使用类型断言可能隐藏类型错误
- **风险等级**: 低
- **建议**: 完善类型定义，减少断言

#### 16. ESLint 规则被绕过
- **位置**: `GeoGebraViewer.tsx:169`
- **问题描述**: `// eslint-disable-next-line react-hooks/exhaustive-deps`
- **风险等级**: 低
- **影响**: 可能隐藏依赖问题
- **建议**: 审查依赖数组，移除禁用注释

#### 17. 空 catch 块
- **位置**: `page.tsx:112-114`
- **问题描述**: `catch { // 忽略解析错误 }`
- **验证结果**: 该位置代码并非 catch 块，而是函数调用参数。真正的 catch 块在 149-151 行，有完整的错误处理逻辑
- **风险等级**: 无

#### 18. localStorage 无容量限制
- **位置**: `utils.ts:43-44`
- **问题描述**: 存储分析结果时没有检查 localStorage 容量
- **风险等级**: 低
- **影响**: 大量数据可能导致存储失败
- **建议**: 添加 try-catch 和容量检查
