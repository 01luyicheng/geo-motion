# GeoMotion 功能规格

## 项目概述

GeoMotion 是一个面向教育场景的AI工具，帮助教师和学生更直观地理解和讲解几何题目。

**核心功能：**
1. 几何题 → 动图：拍照上传几何题目，自动生成解题动画
2. 草图 → 精确图：输入题目文本+手绘草图，生成印刷级精确图形

**目标用户：** 中小学数学教师、学生、教辅资料编撰者

---

## 目标用户与使用场景

### 用户分层

1. 教师（核心用户）
   - 诉求：备课快、课堂演示清晰、可复用。
2. 学生（高频用户）
   - 诉求：看懂图形如何构造、理解解题思路。
3. 内容创作者/教研人员（扩展用户）
   - 诉求：批量生产高质量几何素材并便捷复用。

### 典型场景

1. 课堂演示：教师上传题图，结果页逐步播放作图命令并拖拽验证性质。
2. 作业复盘：学生上传难题，查看已知条件/目标/步骤，单步理解推理过程。
3. 素材生产：教研人员根据文本和草图生成标准化图形，用于讲义与试卷。

### 当前主要痛点

1. 分享能力受限：结果默认保存在本地，跨设备分享不可用。
2. 上传质量波动：缺少拍照引导和上传前裁剪压缩，识别稳定性受输入质量影响。
3. 过程透明度不够：用户很难判断模型推理、语法校验、渲染各阶段状态。
4. 复用效率不足：缺少历史记录中心，旧结果检索成本高。

### 体验目标（建议指标）

1. 首次可用成功率（上传后成功进入结果页） >= 90%。
2. 跨设备分享成功率 >= 95%。
3. 平均首屏生成耗时（含流式首字节） <= 15 秒。
4. 历史结果二次复用率（7天内）提升 30%+。

## AI 响应与前端交互重写规范（2026-03）

为提升等待体验、可恢复性和可分享性，在现有功能基础上新增以下规范。

### 1) 统一事件流协议（SSE v2）

首页与 API 的交互从“纯文本 chunk”升级为“结构化事件流”，每条事件至少包含：

1. `requestId`: 请求唯一标识
2. `seq`: 事件序号（用于乱序/重复保护）
3. `phase`: 当前阶段
4. `event`: 事件类型
5. `payload`: 事件数据
6. `ts`: 服务端时间戳

推荐事件类型：

1. `lifecycle.started`: 请求已受理
2. `content.delta`: 文本增量
3. `command.delta`: GeoGebra 命令增量
4. `validation.result`: 语法校验结果
5. `recovery.attempt`: 自动修复尝试
6. `recovery.result`: 修复结果
7. `lifecycle.done`: 正常结束
8. `lifecycle.error`: 异常结束

兼容策略：旧格式（仅 `content`）在迁移期继续支持。

### 2) 前端状态机规范

首页请求流程采用单一状态机管理，避免多个布尔状态组合导致竞态：

1. `idle`
2. `submitting`
3. `waiting_first_byte`
4. `streaming`
5. `validating`
6. `recovering`
7. `success`
8. `failed_recoverable`
9. `failed_terminal`
10. `cancelling`
11. `cancelled`

UI 文案、按钮可用性、重试入口必须由状态机派生，不允许分散在多个独立 flag 中。

### 3) 用户可见体验要求

1. 提交后 800ms 内必须出现阶段反馈（排队/生成/校验/修复）。
2. 取消请求后 1 秒内进入 `cancelled`，并停止后续渲染。
3. 错误分层展示：
   - 可恢复错误：提供一键重试或自动修复
   - 不可恢复错误：给出明确下一步操作
4. 实时预览优先消费 `command.delta`，避免全量重复解析。

### 4) 分享能力规范

当前本地存储模式保留，但新增服务端临时分享能力：

1. 分享链接支持跨设备访问
2. 结果设置 TTL（例如 24 小时）
3. 链接过期后返回明确提示
4. 本地存储作为离线兜底

### 5) 验收指标（KPIs）

1. TTFB（提交到首事件）P75 <= 2.5s
2. 首条可读文本时间 P75 <= 3.0s
3. 取消生效率 >= 98%
4. 失败恢复率 >= 80%
5. 800ms 内阶段反馈覆盖率 >= 98%

### 6) 分阶段实施计划

1. 第一期（1-2 周）: 引入 SSE v2 协议、埋点与兼容层
2. 第二期（1-2 周）: 前端状态机重构，统一取消/重试/修复链路
3. 第三期（1-2 周）: 命令增量渲染与服务端分享能力上线

## 功能1: 几何题 → 动图

### 输入
- 几何题目的图片（拍照/上传）

### 处理流程

1. **VLM 分析图片**
   - 识别图形类型（三角形、圆形、四边形等）
   - 生成 GeoGebra 命令
   - 提取已知条件
   - 识别求解目标
   - 给出解题步骤

2. **GeoGebra 渲染**
   - 嵌入 GeoGebra Applet
   - 执行命令生成图形
   - 用户可交互操作

3. **动画播放**
   - 逐行执行 GeoGebra 命令
   - 可控制播放速度、暂停、重置

### 输出
- GeoGebra 命令（可复制到 GeoGebra 使用）
- 可交互的几何图形（支持拖动、缩放、测量）
- SVG 导出（已实现）
- PNG 导出（规划中）

---

## 功能2: 草图 → 精确图

### 输入
- 题目文本描述
- 手绘草图照片（可选）

### 处理流程

1. VLM 理解草图意图和文本条件
2. 生成 GeoGebra 命令

### 输出
- GeoGebra 命令（可复制到 GeoGebra 使用）
- SVG 图形（当前实现）
- PNG 图形（规划中）
